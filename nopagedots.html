Making My First Tweak: NoPageDots

This tutorial is made for iOS 10 so things may not be exactly the same in the future.

Things needed for this tutorial
  • theos installation
  • new project ready for your tweak
  • test device to test your tweak
  • iOS 10 Headers

Step 1: Finding the Class to Hook

The first part of making a tweak is finding the classes to hook. A lot of tutorials skip this step but an easy way of doing this is using Develope.Limneos.net. This has a search bar which makes easy to find which header you are looking for.

The aim of this tweak will be to hide the page dots on the homescreen. After thinking about what you want to do, use Developer.Limneos.net to search for the correct header. The header will be part of SpringBoard as SpringBoard is the framework which your homescreen is part of (Your homescreen uses other frameworks as well but ssshh). I'll give you a hint, the name of this header isn't simply SBPageDots.

The homescreen iOS page dots are part of Page Control and SpringBoard so you can go to your headers are search for anything containing "PageControl" (You can narrow it down to SpringBoard.framework and use CTRL+F to search). If you find a header that you think may contain what you are looking for. Open up that header and look at the methods within it (The -(void)s & -(id), etc

Found It? It's SBIconListPageControl. The code below is an example and each line will be explained

Step 2: The Code

%hook SBIconListPageControl

This hooks the class that we want to modify

-(id)initWithFrame:(CGRect)arg1

This is the method within the class that we want to modify. "arg1" is argument 1 for the method. Modify arguments is a common thing for beginner tweaks. Download the example project of other ways to modify an argument.

{ return nil; }

This line returns "nil" for this class. Meaning arg1 for initWithFrame will return nothing

%end

This ends the %hook

Run make do or make package install to compile the code and install it onto your iPhone.

That's about it for this tutorial. Check out other headers and look for more things to modify.
